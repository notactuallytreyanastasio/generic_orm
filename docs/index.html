Usage

Same API, every language

The ORM compiles to idiomatic code in each target. The API adapts to naming conventions — camelCase in JS, snake_case in Python/Rust, PascalCase in Java/C#. The safety model doesn't change.

import { from, safeIdentifier, changeset, deleteSql,
         SqlBuilder, SqlString } from 'orm';

// Query with WHERE, ORDER BY, LIMIT
const q = from(safeIdentifier("todos"))
  .where(whereFragment)
  .orderBy(safeIdentifier("created_at"), true)
  .limit(25)
  .toSql().toString();

// INSERT via changeset
const cs = changeset(tableDef, params)
  .cast([titleId, completedId])
  .validateRequired([titleId])
  .validateLength(titleId, 1, 255)
  .toInsertSql().toString();

// DELETE by primary key
const d = deleteSql(tableDef, 42).toString();

Security

Five layers that make injection impossible

The ORM doesn't prevent SQL injection by being careful. It prevents it by making unsafe SQL impossible to construct. The type system won't let you.

SafeIdentifier

Table/column names validated against [a-zA-Z_][a-zA-Z0-9_]*. Sealed — no bypass.

SqlPart Hierarchy

Sealed type per data kind. String escaping, integer rendering, NaN → NULL. Exhaustive handling enforced.

SqlBuilder

Structure vs data separated at API level. No appendRaw. Only appendSafe for keywords/identifiers.

Changeset Pipeline

Sealed interface. cast() whitelists fields. Raw params never exposed. Prevents mass assignment.

Query Builder

All structural positions require SafeIdentifier. WHERE accepts only SqlFragment. No raw strings.

Mutation Guards

UPDATE/DELETE refuse to run without WHERE. JOIN keywords sealed. Lock modes hardcoded.

safeIdentifier("users")       // → ValidatedIdentifier ✓
safeIdentifier("users; DROP") // → bubble (rejected at construction)

Background

Built by one person and an AI

This project — the ORM, all 6 apps, the CI pipeline, the security analysis, and this site — was built by a single developer working with Claude. The AI didn't need to know 6 languages. It worked in Temper. The compiler did the rest.

To test whether this matters, we asked an AI to build a comparable ORM in Ruby with the same prompts. Then we ran the same MITRE analysis on both.
