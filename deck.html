SECURITY ARCHITECTURE

5 compile-time defense layers

#LayerCWEWhat It Prevents
1 SafeIdentifier CWE-89 Table/column name injection. [a-zA-Z_][a-zA-Z0-9_]* validated, sealed class not exported.
2 Typed SqlPart CWE-89 Value injection. Sealed interface — SqlString escapes quotes, SqlInt32 renders bare integers. Exhaustive dispatch.
3 SqlBuilder CWE-89 Structural injection. appendSafe only accepts validated identifiers + hardcoded literals. No appendRaw.
4 Changeset CWE-915 Mass assignment. Sealed interface, cast() requires List<SafeIdentifier>. Raw params never exposed.
5 Query Builder CWE-400 Unbounded queries. safeToSql(limit) enforces result set bounds. WHERE accepts only SqlFragment.

These defenses are structural, not advisory. The type system makes it impossible to construct unsafe SQL without deliberately circumventing the API.
